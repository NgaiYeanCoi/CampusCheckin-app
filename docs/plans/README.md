# docs/plans

本目录用于保存计划模式相关记录。只有当任务进入计划模式、用户明确要求先做计划，或需要产出正式实施计划时，才需要在本目录创建或更新计划文件。

普通问答、简单解释、单步修复、环境检查、直接执行型小任务不强制记录。

## 命名规则

```text
YYYY-MM-DD-task-name-plan.md
```

示例：

```text
2026-05-17-server-api-integration-plan.md
2026-05-17-android-login-page-plan.md
2026-05-17-database-schema-update-plan.md
```

## 记录内容

每个计划文件至少包含：

- 任务标题
- 当前背景
- 目标
- 计划改动
- 涉及文件
- 验证方式
- 当前状态

## 使用规则

- 全新的计划模式任务创建新计划。
- 继续同一计划任务时更新已有计划。
- 计划内容或实现方案发生明显变化时同步更新计划。
- 计划文件应服务于后续对话恢复上下文，避免只靠聊天记录回忆。
